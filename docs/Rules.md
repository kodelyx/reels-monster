# 📏 Rules.md — Reels-Monster ke pakke niyam

Ye project ke permanent rules hain. Koi bhi kaam karne se pehle ye follow karo.

---

## ✅ RULE 1 — Har update ke baad VERIFY zaroori

> Koi bhi stage/code **update** ya **bada change** karo → usko chalao (test) → output ko **AI examiner** se jachwao. Verify ke bina kaam **PASS nahi** hoga.

**Kaise verify karna hai:**
1. Jo stage badla, usko chalao (real output banao).
2. Output ko strict AI examiner ko do — rating **10 me se** maango.
3. Examiner ko bolo: **chaploosi mat karo, "ha me ha" mat milao, sakht examiner ki tarah asli dikkat batao.**
4. Rating + problems dekho. Kharab hai → dobara sudhaaro, phir verify.

**Pass line:** verify nahi hua = kaam adhura = merge/commit nahi.

---

## ✅ RULE 2 — Ek baar me ek hi part

> Ek time pe sirf ek stage/part chhuo. Baaki code untouched. Change ke baad commit — kuch toote to `git checkout` se wapas.
